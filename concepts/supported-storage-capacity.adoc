= Keystone支援的儲存容量
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
NetApp Keystone STaaS 服務支援多種類型的儲存容量。了解這些不同的容量術語可以幫助您使用Keystone。



[[logical-capacity]]
== 邏輯容量

這是客戶放置在Keystone基礎架構上的資料。所有Keystone容量均指邏輯容量。例如，如果Keystone基礎架構上儲存了 1 TiB 的文件，則至少應購買 1 TiB 的容量。



[[committed-capacity]]
== 承諾產能

訂閱期間每月計費的最小邏輯容量：

* 容量致力於每個效能服務等級。
* 承諾容量和額外的服務等級可以在期限內添加。




[[changes-to-committed-capacity]]
=== 承諾產能的變化

在訂閱期限內，您可以更改承諾的容量。但是，有一定的前提條件：

* 根據特定條件，承諾容量可能會減少。有關信息，請參閱link:../concepts/capacity-requirements.html["產能削減"]。
* 承諾容量不能在您的訂閱到期前 90 天增加，除非訂閱期限額外續約 12 個月。
* 您可以透過控制台或向您的Keystone Success Manager (KSM) 要求變更已承諾的容量。有關申請更改的信息，請參閱 link:../concepts/support-services.html["NetApp Keystone支持"]。




[[consumed-capacity]]
== 已消耗容量

已使用容量是指服務目前正在使用的容量（以 TiB 儲存為單位）。根據資料儲存類型的不同，其計算方式也不同：

* *統一儲存或區塊最佳化儲存設備*：已使用容量根據訂購流程中選擇的容量類型（邏輯容量或實體容量）計算。計算按效能服務層級執行個體執行。
+
.. *邏輯容量*：它是以下各項的總和：
+
*** 在儲存陣列資料效率之前，計量邏輯容量用於儲存所有執行個體和類型的客戶資料，例如副本、鏡像副本、版本和複製。
*** 用於儲存快照和某些複製的中繼資料及差異資料的實體容量。
*** 任何完整佈建的實體容量。


.. *實體容量*：它是下列各項的總和：
+
*** 計量實體容量、儲存陣列資料效率之後、用於儲存所有執行個體和類型的客戶資料、例如複本、鏡射複本、版本和複本。
*** 用於儲存快照中繼資料和差異資料的實體容量。
*** 任何完整佈建的實體容量。




* *物件儲存設備*：已使用容量是指用於儲存所有節點上所有執行個體和類型客戶資料的計量實體容量的總和。此計算基於已設定的資訊生命週期管理 （ILM） 原則。
* *Cloud Volumes ONTAP*：已消耗容量以所有 Cloud Volumes ONTAP 磁碟區的計量配置邏輯容量計算。




[[burst-capacity]]
== 突發容量

NetApp Keystone服務讓您能夠在服務等級承諾容量的基礎上使用額外容量。這被稱為突發容量使用。

請注意以下幾點：

* 突發容量是在Keystone協定中約定的。它通常設定為高於承諾容量的 20%，並按照與承諾容量相同的費率收費。
* 突發容量可彈性消耗，並按日平均消耗量收費。




[[billed-capacity]]
== 計費容量

每月費用 = (承諾容量 [TiB] * 承諾費率 [$/TiB]) + (每日平均預置突發容量 [TiB] * 突發費率 [$/TiB])。每月帳單包含基於承諾容量的最低費用。

根據每日平均突發容量消耗，每月帳單會超出最低費用。
